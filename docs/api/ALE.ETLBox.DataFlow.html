Namespace ALE.ETLBox.DataFlow

Classes

BlockTransformation<TInput>

A block transformation will wait for all data to be loaded into the buffer before the transformation is applied. After all data is in the buffer, the transformation is execution and the result posted into the targets.

CSVSource

Reads data from a csv source. While reading the data from the file, data is also asnychronously posted into the targets. CSVSource as a nongeneric type always return a string array as output. If you need typed output, use the CSVSource<TOutput> object instead.

CSVSource<TOutput>

Reads data from a csv source. While reading the data from the file, data is also asnychronously posted into the targets. Data is read a as string from the source and dynamically converted into the corresponding data format.

CustomDestination<TInput>

Define your own destination block.

CustomSource<TOutput>

Define your own source block.

DBDestination

A database destination defines a table where data from the flow is inserted. Inserts are done in batches (using Bulk insert). The DBDestination access a string array as input type. If you need other data types, use the generic DBDestination instead.

DBDestination<TInput>

A database destination defines a table where data from the flow is inserted. Inserts are done in batches (using Bulk insert).

DBSource<TOutput>

A database source defines either a table or sql query that returns data from a database. While reading the result set or the table, data is asnychronously posted into the targets.

ExcelRange

ExcelSource<TOutput>

Reads data from a excel source. While reading the data from the file, data is also asnychronously posted into the targets. You can define a sheet name and a range - only the data in the specified sheet and range is read. Otherwise, all data in all sheets will be processed.

Lookup<TTransformationInput, TTransformationOutput, TSourceOutput>

A lookup task - data from the input can be enriched with data retrieved from the lookup source. The result is then posted into the output.

MergeJoin<TInput1, TInput2, TOutput>

Will join data from the two inputs into one output - on a row by row base. Make sure both inputs are sorted or in the right order.

MergeJoinTarget<TInput>

Multicast<TInput>

A multicast duplicates data from the input into two outputs.

RowTransformation

Transforms the data row-by-row with the help of the transformation function. The non generic RowTransformation accepts a string array as input and returns a string array as output. If you need other data types, use the generic RowTransformation instead.

RowTransformation<TInput, TOutput>

Transforms the data row-by-row with the help of the transformation function.

Sort<TInput>

Sort the input with the given sort function.
